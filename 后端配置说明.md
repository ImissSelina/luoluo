# 后端配置说明

这个项目使用 `server.js` 作为本地后端，默认地址：

```text
http://127.0.0.1:6066
```

后端负责两件事：

1. 托管公开静态文件：`index.html`、`styles.css`、`app.js`、`assets/`
2. 代理 AI API 请求：浏览器只请求 `/api/...`，API Key 只在服务器读取，不暴露给前端

## 启动

双击：

```text
启动网站-6066.bat
```

或在项目目录运行：

```powershell
node server.js
```

## 配置文件

真实配置放在：

```text
.env
```

第一次配置时可以复制 `.env.example`，再填入真实 Key。

不要把 API Key 写进 `app.js`、`index.html` 或任何前端文件。

## 聊天配置

```text
GPT_API_KEY=你的密钥
GPT_CHAT_URL=https://你的接口域名/v1/chat/completions
GPT_MODEL=真实模型名

CLAUDE_API_KEY=你的密钥
CLAUDE_CHAT_URL=https://你的接口域名/v1/chat/completions
CLAUDE_MODEL=真实模型名

MIMO_API_KEY=你的密钥
MIMO_CHAT_URL=https://你的接口域名/v1/chat/completions
MIMO_MODEL=真实模型名
```

## 图片配置

推荐使用当前格式：

```text
CHATGPTIMAGE2_API_URL=https://api.xinnanz.cn/v1
CHATGPTIMAGE2_API_KEY=你的图片密钥
CHATGPTIMAGE2_MODEL=gpt-image-2
CHATGPTIMAGE2_MODE=multipart
CHATGPTIMAGE2_EDIT_PATH=images/edits
CHATGPTIMAGE2_GENERATE_PATH=images/generations
REQUEST_TIMEOUT=1200
```

后端也兼容旧格式：

```text
GPTIMAGE2_API_KEY=你的图片密钥
GPTIMAGE2_EDIT_URL=https://api.xinnanz.cn/v1/images/edits
GPTIMAGE2_GENERATE_URL=https://api.xinnanz.cn/v1/images/generations
GPTIMAGE2_IMITATE_URL=https://api.xinnanz.cn/v1/images/edits
```

## 图片请求逻辑

- 不上传参考图：走 `/images/generations`，使用 JSON 文生图
- 上传原图或模板图：走 `/images/edits`，使用 multipart 传图
- 多图上传不是多次请求，而是在一次 multipart 请求里重复追加多个 `image` 字段
- 模板下拉选项会被转换成 prompt 里的风格描述，不会把本地字段直接丢给上游

## 安全边界

浏览器只能访问公开静态文件和 `/api/...` 接口。

以下路径应始终返回 `404`：

```text
/.env
/server.js
```

`.env` 已被 `.gitignore` 忽略，不要上传或公开分享。
